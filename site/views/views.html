Views

GoJS automatically handles retrieving, caching and rendering of HTML views.

Setting Views

You can use the view property of an action to define which view is retrieved and rendered. The location of the views can be customized in the config. 

        //Set the view location in the config
        Go.config({
            viewRoute: "/app/views/{key}.html",
        }); 

        //Set view property in actions
        var InfoController = new Controller({

            getInfo: new Action({           
                view: "about",              //Retrieve view from '/app/views/about.html'
                updater: "content"          //Render view in main content area
            })

        });

Preloading views

You can also preload views. This is particularly useful when multiple child views are being called. 

        var InfoController = new Controller({

            getInfo: new Action({           
                require: {
                    views: "childview1, childview2",    //Load these views
                },
                view: "mainview"                        //Load and use as main view
            })

        });
