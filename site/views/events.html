Events

Each time an action is run, a set of events occur.

  1. Run - An action is being executed.
  2. Request - A server request is being made.
  3. Response - A successful server response was recieved.
  4. RequestError - A server response failed. A 404 for example.
  5. Success - A server operation was successful... { success: true }
  6. Error - A server operation returned an error... { error: "A error occurred" } or { errors: [] }
  7. Render - A view is being rendered.
  8. Update - An element is being updated in the DOM. See updates for details.
  9. Activate - An element is being "activated". Triggered on page load and after dynamic updates.
  10. Complete - The event is finished.

Action Event Handlers

Event handlers can be added to a specific action. 

        var TodoController = new Controller("todo", {
        
            add: new Action({
                run: function() { ... }
                request: function() { ... }
                response: function() { ... }
                requestError: function() { ... }
                success: function() { ... }
                error: function() { ... }
                render: function() { ... }
                update: function() { ... }
                activate: function() { ... }
                complete: function() { ... }
            })

        });

Global Event Handlers

Event handlers can also be run globally on every action using the Go.on() method. The is an extremely powerful feature that allows you to add functionally to an entire application with very little code.

The following example adds a placeholder function to each input element with a "placeholder" attribute. The activate event is triggered during page load and every time an update is made.

Global Placeholder Shim 
        Go.on("activate", function(e) {
            e.element.find("input[placeholder]").each(function() {
                $(this).placeholder();  //Using custom placeholder plugin
            })
        });
