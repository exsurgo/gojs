Persistent Objects

GoJS features automatic storage and retrieval of objects from local or session storage.

Using the Go.persist(name, value) method, objects are marked for persistence. In the example below, a new object myObject" is created under the namespace "app", with an set of initial values. When the page is unloaded, the object is automatically stored in session or local storage. On the next page load, the object is automatically retrieved and loaded into memory. 

        Go.persist("app.myObject", {
            someNumber: 124,
            someString: "My Persistent Object",
            someBool: true,
            someArray: [...]
        })

Basic Storage

GoJS also provides several helpful methods for working with session/local storage. The Go.store(key, value, permanent) method will serialize and store some value, while the Go.retrieve(key) method will retrieve the value. 

        
        //Store in session storage
        Go.store("my-object-key", {...});

        //Store in local storage
        Go.store("my-object-key", {...}, true);

        //Retrieve from either session or local storage
        Go.retrieve("my-object-key");

There are a couple benefits in using GoJS for storage over sessionStorage/localStorage directlly. First, session/local storage only supports strings, while GoJS handles the serialization automatically. Second, the retrieve method searches both local and session storages with a single call.
